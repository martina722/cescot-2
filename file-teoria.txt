Strumenti per la gestione di progetti
Gestire un progetto = avere strumenti che permettono di scomporre prodotto, assegnare task, gestire presenze, ... Si possono quindi utilizzare vari strumenti software, ognuno di essi è specializzato in un ambito d'uso specifico, la maggior parte di loro sono collaborativi; si dividono in base al framework organizzativo utilizzato:
- kanban, rappresenta azioni (task) scomposte in elementi (azioni)
- canva, rappresenta scomposizioni non organizzate
- Ishikawa, rappresenta relazioni causali (causa/effetto)
Esistono diversi software in grado di modellare processi (framework), in base al software è possibile utilizzare uno o più framework teorici e sono disponibili automatismi (promemoria per le scadenze, notifiche automatiche, ...).

Excel
Excel consente di rappresentare (per sua natura) tabelle, tramite esse è possibile rappresentare processi con scadenze, obiettivi, responsabili, ... Excel è molto diffuso e flessibile, è disponibile gratuitamente, ma non offre automatismi per la gestione dei processi.

Loop
Loop (loop.microsoft.com) implementa un canva collaborativo, tramite questo programma è possibile modellare canva di qualsiasi tipo, inserire elementi strutturati (tabelle, link, ...), collaborare con altri utenti. Loop invia notifiche automatiche quando viene menzionato un utente all'interno del canva (eurosoftlab.com).
Loop consente di inserire elementi programmatici: viene creato uno spazio (nel canva) nel quale è possibile scrivere codice di vario tipo (LaTex, PHP, ...) per generare contenuti strutturati.
WhiteBoard
Microsoft Whiteboard (whiteboard.microsoft.com) è un canva collaborativo (analogo a Loop) che non possiede automatismi (non invia notifiche), ma prevede modelli per i framework teorici più diffusi.

Jira
Jira (www.atlassian.com/software/jira) è un software per la gestione di progetti basato su framework Kanban. Il sistema consente di condividere i progetti con altri utenti, invia notifiche automatiche, consente di visualizzare i progetti in diversi modi (tabella, Gantt, ...).

Planner
Planner (planner.cloud.microsoft) è un software a pagamento disponibile con licenza Microsoft 365 aziendale che consente di gestire progetti basandosi sul framework Kanban. Planner consente di suddividere un progetto in task e di suddividere le task in azioni, ognuna può essere associata ad un utente. Planner invia notifiche automatiche e gestisce lo stato di ogni azione.

Frontend/DB
Quando si sviluppano applicazioni web è necessario archiviare i dati in modo da consentire di costruire pagine frontend dinamiche. Una pagina dinamica non ha dei contenuti noti in fase di implementazione: i contenuti vengono caricati da un databse in fase di utilizzo. Per generare pagine con contenuti dinamici è necessario leggere i contenuti da un databse (o da un'altra sorgente), questo consente di suddividere la gestione di un'applicazione web in:
- programmazione del frontend (definizione della struttura e del funzionamento utilizzando HTML, PHP, SQL, css, JS)
- gestione dei contenuti (databse utilizzando SQL)
Una pagina dinamica esegue delle query SQL per caricare i contenuti e monta i risultati all'interno del codice HTML, per farlo deve connettersi al databse, questo avviene tramite librerie PHP.
